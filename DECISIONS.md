# Decisions

最終更新: 2026-03-16

## 決定事項

### 1. YouTube は直接売買判断に使わない

- YouTube 単独で `BUY` は出さない
- まず watchlist を作る

### 2. 基本フローは固定する

運用フロー:

1. YouTube 多数決
2. 候補銘柄化
3. AI 提案
4. 別データ検証
5. 最終判断

### 3. `BUY` は価格確認が必要

- watchlist 銘柄は価格確認が取れるまで `BUY` にしない
- 条件不足なら `WATCH` に落とす

### 4. 固定チャンネルを重視する

- 検索だけに頼らない
- 固定チャンネルを基準点として持つ
- 検索は補完として使う

### 5. Discord 通知には行動ラベルを出す

- `BUY`
- `WATCH`
- `AVOID`
- `NO SIGNAL`

### 6. Discord で由来を見える化する

- 固定チャンネル由来か
- 検索由来か
- 上位チャンネルは何か

### 7. AI は提案役に留める

- AI が候補を出す
- ルールと別データで確認する
- AI に全面委任しない

### 8. ローカルの重さは許容する

- `youtube` は GitHub Actions で安定運用する
- ローカル同期実行の重さは現時点では致命傷扱いしない

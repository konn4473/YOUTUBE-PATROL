# Operation Guide

## 目的
このプロジェクトは、YouTube を見てそのまま買うためのものではありません。

基本方針は次です。

1. YouTube の多数決で話題を拾う
2. 候補銘柄を作る
3. AI が候補を提案する
4. 別データで検証する
5. 条件がそろったときだけ買う

つまり、YouTube は「監視候補抽出エンジン」、AI は「候補提案エンジン」として使います。

## 基本フロー

### 1. YouTube 多数決

目的:

- 今何が話題かを把握する
- 複数チャンネルで共通して語られているテーマを拾う

見るもの:

- 言及回数
- チャンネル数
- グループ数
- sentiment

この段階では `BUY` を出しません。

### 2. 候補銘柄化

YouTube で出たテーマから候補銘柄を作ります。

例:

- 半導体 → `8035`, `6920`, `6857`
- 防衛 → `7011`, `7012`, `7013`
- 円安 → `7203`, `6758`, `6902`

この段階の出力は watchlist です。

### 3. AI 提案

AI は watchlist・ニュース・市場データを見て、次のような提案を返します。

- `BUY`
- `WATCH`
- `AVOID`
- `NO SIGNAL`

ただし、この `BUY` はまだ仮の提案です。
この時点では、まだそのまま注文しません。

### 4. 別データ検証

AI が提案した銘柄を、次のデータで検証します。

確認項目:

1. 価格
   - 前日比プラスか
   - ブレイク気味か

2. 出来高
   - 普段より増えているか

3. ニュース
   - 追い風ニュースがあるか

4. 地合い
   - 日経や米国株が極端に悪くないか

5. リスク
   - 急騰後の高値掴みにならないか

現在の実装では、特に watchlist 銘柄について `価格確認` を重視します。

### 5. 最終判定

最終判定は council に任せます。

考え方:

- `BUY`
  - AI 提案あり
  - 価格確認あり
  - 裏付けも悪くない

- `WATCH`
  - 話題にはなっている
  - ただし価格確認や裏付けが足りない

- `AVOID`
  - 弱気テーマ
  - 悪材料優勢
  - 地合い悪化

- `NO SIGNAL`
  - 情報不足
  - 一致が弱い

## Discord 通知の見方

### YouTube 通知

主に見るもの:

- `Action`
- `Themes`
- `Candidates`
- `Video`
- `Trading note`

見方:

- `WATCH`
  - 監視候補として見る
- `AVOID`
  - 弱気寄りなので見送り寄り
- `NO SIGNAL`
  - 何もしない

YouTube 通知だけで買う判断はしません。

### main patrol 通知

主に見るもの:

- `Action`
- `Summary`
- `AI proposal`
- `News`
- `Decision`
- `Trading note`

こちらが最終判断に近い通知です。

## このプロジェクトでの役割分担

- YouTube
  - テーマ抽出
  - watchlist 作成

- AI
  - 候補銘柄提案
  - `BUY / WATCH / AVOID / NO SIGNAL` の一次提案

- ニュース / 価格
  - 追認
  - 条件確認

- council
  - 最終判断

## 実務上の注意

1. YouTube は遅いことがある
   - 話題になった時点で価格が先に動いていることがあります

2. 多数決だけで買わない
   - 人気化した後の高値掴みを避けるためです

3. AI の `BUY` もそのまま買わない
   - 価格確認やニュースの裏付けを見てから動きます

4. `WATCH` は「次に見るべき候補」
   - まだ買いではありません

5. `NO SIGNAL` は何もしない
   - 無理にエントリーしない方が良いです

## 推奨ルール

迷ったときは次で考えると分かりやすいです。

1. YouTube は入口
2. AI は候補提案
3. ニュースは裏付け
4. 価格は実行条件
5. 最終判断は保守的にする
